Proyecto de Análisis de eCommerce
Este repositorio contiene el desarrollo técnico realizado para el análisis integral de una plataforma de comercio electrónico. El flujo de trabajo abarca desde la limpieza de datos hasta la implementación de modelos predictivos.

Estructura de los Scripts
Los archivos han sido numerados siguiendo el orden lógico de ejecución del proyecto:

01_limpieza_eda.py: Procesamiento de los datos brutos y análisis exploratorio inicial para asegurar la calidad de la información.

02_modelado_de_datos.py: Integración de múltiples fuentes de datos (pedidos, productos, usuarios y logística) en una tabla maestra consolidada.

03_analisis_exploratorio.py: Exploración de indicadores clave (KPIs) como márgenes de beneficio, estacionalidad y eficiencia logística para validar la lógica de negocio.

04_segmentacion_clientes.py: Aplicación de técnicas de Clustering (K-Means) para la identificación de perfiles de usuario según su comportamiento.

05_modelo_fuga.py: Desarrollo de un modelo de Machine Learning para predecir el riesgo de abandono de los clientes.

Nota sobre los datos
Por motivos de privacidad y debido al tamaño de los archivos, los datasets originales y procesados no se incluyen en este repositorio. Los scripts se presentan como muestra de la metodología técnica y la lógica aplicada en cada fase.

Para consultar el análisis visual completo y las conclusiones de negocio, por favor vuelve a la página de este proyecto en Notion.
